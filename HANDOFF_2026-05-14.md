## Bring-Up Path: Next Steps (High Priority)

### 1. INA Current Mapping Debug ⏳ **BLOCKER**
**Why**: Current readings don't match known loads (10Ω @ 3.3V reads 18mA instead of ~327mA)

**Procedure**:
1. Extract channel-to-shunt truth table from HAT schematic (which INA channel sees which shunt?)
2. Meter actual shunt resistor values on board (verify 20mΩ/200mΩ/10mΩ assumptions)
3. Apply single known load to one rail; capture INA readings; correlate to identify correct channel
4. Repeat for each rail until mapping confirmed
5. Update firmware mappings in `shuntOhmsForTap()`

**Effort**: 1–2 hours bench work + documentation

**Success Criteria**: INA current readings within ±5% of known loads

### 2. Adjustable Rail Feedback Investigation ⏳
**Why**: Adjustable rail reading same voltage as 3.3V reference (should be tunable via ISET pot)

**Procedure**:
1. Multimeter: Check continuity across feedback divider network (look for open circuit)
2. Verify potentiometer (ISET) is wired and functional (measure resistance across pot legs)
3. Measure feedback network voltage; compare to potentiometer output
4. If feedback present, investigate why output locked at 3.3V
5. Check for shorts between feedback and adjacent traces

**Effort**: 30 minutes bench work

**Success Criteria**: Adjustable rail responds to ISET potentiometer (±0.1V per turn)

### 3. Load Stability Testing (After INA Debug)
**Procedure**:
1. Apply steady 100mA load to 5V rail for 5 minutes; log input current stability
2. Apply steady 50mA load to 3.3V rail for 5 minutes; log voltage ripple
3. Check for thermal runaway: temperature stable or trending up?
4. Record baseline for long-term drift testing (compare to next session)

**Effort**: 30 minutes once INA mapping complete

**Success Criteria**: Rails stable ±1% over 5-minute window; no thermal climb

### 4. Protection Mechanism Validation (After Stability Confirmed)
**Procedure**:
1. Intentional short-circuit on 5V rail (safe conditions; verify fuse trips)
2. Verify HICCUP mode recovery (power cycles if present)
3. Log current-limit threshold for each rail
4. Check for latch-off (auto-recovery vs. manual reset required)

**Effort**: 1 hour bench work

**Success Criteria**: Fuses blow cleanly; no board damage; recovery as designed

### 5. Multi-Rail Simultaneous Load Testing (After Individual Rails Validated)
**Procedure**:
1. Apply 50mA to 5V + 30mA to 3.3V simultaneously
2. Measure input current and check for regulation coupling
3. Monitor for any voltage dips or cross-rail interference
4. Compare total input current to sum of individual rail currents

**Effort**: 30 minutes

**Success Criteria**: No cross-rail coupling; regulation independent per rail
