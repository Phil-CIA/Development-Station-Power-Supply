# Handoff — 2026-05-30 — Wired UART link WORKING

## TL;DR

The HAT↔CrowPanel **wired UART link is operational** through the CrowPanel
CH486F mux. **ESP-NOW remains active as a redundant backup.** Both transports
publish the same 10-byte telemetry frame in parallel every ~5 s. This is the
new known-good baseline for the C6-HAT + CrowPanel-4.3 display path.

A future Rev 2 HAT can swap the ESP32-C6 for an STM32 (Blue Pill) and keep
the same wiring + frame format — UART will continue to work without any
wireless fallback.

---

## What changed this session

### Root cause of the long-running "wired UART dead" bug

S3-side `Serial1` was started with **RX and TX pins swapped** relative to
Elecrow's own reference firmware.

Authoritative reference (Elecrow CrowPanel-Advance-4.3 GitHub repo,
`example/V1.1_and_V1.2/arduino/lesson-09/zigbee_7.0/zigbee_7.0.ino`):

```cpp
#define SERIAL1_RX 19   // Serial port 1 receives pins
#define SERIAL1_TX 20   // Serial port 1 Send pins
Serial1.begin(115200, SERIAL_8N1, SERIAL1_RX, SERIAL1_TX);
```

So from the **S3's perspective on the HY2.0 UART1-OUT connector**:

| S3 pin | Direction        | Goes to (via CH486F U9) |
|--------|------------------|--------------------------|
| IO19   | UART1 **RX**     | HAT GPIO7 (HAT TX)       |
| IO20   | UART1 **TX**     | HAT GPIO0 (HAT RX)       |

The CH486F passes 115200 8N1 TTL just fine — earlier theory that it was
"differential-USB-only" was wrong, contradicted by Elecrow's shipping
example.

### Fix applied

`crowpanel-43-bringup/src/disp_link_slave.cpp`:

```cpp
constexpr int kUartRxPin = 19;   // was 20
constexpr int kUartTxPin = 20;   // was 19
```

Log banner now correctly reads:

```
disp_link_slave: UART1 listening @115200 baud RX=IO19 TX=IO20 (K1 must be 0,1)
```

### Live evidence the link works

CrowPanel serial monitor, post-fix:

```
rx frames=238 errs=0 i2c=119 seq=191 V12=11648 mV I12=91 mA age=573 ms uartB=1190
rx frames=240 errs=0 i2c=120 seq=192 V12=11648 mV I12=93 mA age=475 ms uartB=1200
rx frames=242 errs=0 i2c=121 seq=193 V12=11648 mV I12=93 mA age=378 ms uartB=1210
rx frames=244 errs=0 i2c=122 seq=194 V12=11648 mV I12=93 mA age=279 ms uartB=1220
```

- `uartB` (raw Serial1 RX bytes) +10 per HAT publish cycle ✅
- `i2c` (valid 10-byte UART frames, name retained for backward-compat in
  the snapshot struct) +1 per cycle ✅
- `rx frames` (ESP-NOW backup) still +2 per cycle ✅

The LCD bottom-left stats label now displays the same counters live:
`rx=N err=N seq=N uartB=N i2c=N`.

## Required physical configuration (CrowPanel)

| Item | Setting |
|------|---------|
| K1 DIP S0 | **1 (ON)** |
| K1 DIP S1 | **0 (OFF)** |
| Per silkscreen | "To use UART1_OUT, please switch S1 and S0 to 0 and 1" |
| HY2.0 UART1-OUT pin "RX" | wired to HAT GPIO7 (HAT TX) |
| HY2.0 UART1-OUT pin "TX" | wired to HAT GPIO0 (HAT RX) |
| GND | common between both boards |

CH486F enable (EN#) is hard-pulled high by R61 (10 K); no firmware mux
control is required, and Elecrow's own examples confirm this.

## Firmware state

### HAT — ESP32-C6 (COM11, env `hat_c6_i2c_probe`)

- `src/disp_link.cpp` — Serial1 on GPIO0(RX)/GPIO7(TX), 115200 8N1.
- `publishTelemetry()` writes the frame on Serial1 **and** broadcasts via
  ESP-NOW each call. (Per-cycle diagnostic prints were removed today; only
  failures are logged.)
- `src/i2c_scanner.cpp` — bit-bang INA3221 reader on GPIO5/6, reads 12 V
  rail every ~5 s, drives `publishTelemetry()`.

### CrowPanel — ESP32-S3 (COM12, env `crowpanel43`, project in
`crowpanel-43-bringup/`)

- `src/disp_link_slave.cpp` — Serial1 on IO19(RX)/IO20(TX), USB-Serial-JTAG
  pad-enable bit cleared + `gpio_reset_pin(19/20)` **before** `Serial1.begin`.
  Native USB CDC console (COM12) continues to work for monitoring — the
  cleared bit only releases the pads from being actively driven so Serial1
  can listen.
- `src/main.cpp` — LVGL 8.3.11 dashboard. Bottom-left label now shows
  `rx err seq uartB i2c` counters refreshed on any change. Diagnostic
  `PROBE <pin> [ms]` command still available in the serial REPL.
- ESP-NOW recv callback continues to populate the same `Telemetry` snapshot
  alongside the UART SOF state machine.

### Frame format (unchanged)

```
[0xAA][0x55][LEN=0x06]['T'][seq][v_lo][v_hi][i_lo][i_hi][CRC8]
```

CRC-8 polynomial 0x07, init 0x00, computed over bytes [LEN .. i_hi].

## Things deliberately NOT done

- The HAT's RX path on Serial1 is wired but unused — the HAT does not parse
  any inbound frames yet. CrowPanel→HAT direction is reserved for future
  commands (set rail, query, ack, etc.).
- The duplicate transports (UART + ESP-NOW) are intentionally both left on.
  ESP-NOW will be the only path that survives a future STM32 swap-out only
  in the sense that it won't be available; the wired UART is the primary
  long-term link.
- `disp_link_slave::Telemetry::i2c_rx_count` field name is historical; it
  now counts valid UART frames. Renaming it touches `main.cpp` and several
  printfs — left alone for now to keep the diff small.
