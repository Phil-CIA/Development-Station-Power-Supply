### Key LVGL Integration Notes (required for future changes)

**GCC `__has_include` silent-fail (bug #80753):**
`lv_conf.h` in `include/` is NOT reliably seen by library compilation units. Critical settings must be in `build_flags` as `-D` flags, not only in `lv_conf.h`.

Required `build_flags` in `platformio.ini`:
```ini
-DLV_CONF_INCLUDE_SIMPLE
-DLV_FONT_MONTSERRAT_12=1
-DLV_FONT_MONTSERRAT_16=1
-DLV_FONT_MONTSERRAT_20=1
-DLV_FONT_MONTSERRAT_28=1
-DLV_FONT_MONTSERRAT_48=1
```

**Flush callback — use `pushImage`, not `pushImageDMA`:**
```cpp
void lvgl_flush_cb(lv_disp_drv_t* drv, const lv_area_t* area, lv_color_t* color_p) {
  display.startWrite();
  display.pushImage(area->x1, area->y1,
                    area->x2 - area->x1 + 1,
                    area->y2 - area->y1 + 1,
                    (lgfx::rgb565_t*)color_p);
  display.endWrite();
  lv_disp_flush_ready(drv);
}
```
Do NOT leave a `startWrite()` open from `setup()` and rely on it inside `flush_cb`. Flush_cb manages its own write context.

**Manual tick (LV_TICK_CUSTOM=0, default):**
```cpp
// in loop():
lv_tick_inc(millis() - lv_last_tick_ms);
lv_last_tick_ms = millis();
lv_timer_handler();
```

**PSRAM framebuffers:**
Two 800×480×2B buffers allocated with `heap_caps_malloc(MALLOC_CAP_SPIRAM | MALLOC_CAP_8BIT)`.
`lv_conf.h` leaves `LV_MEM_CUSTOM=0` (default heap for LVGL objects); only the draw buffers are in PSRAM.

## Next Session Goal: UART HAT↔CrowPanel Debug

### Background

The UART wired link was investigated but abandoned in favor of ESP-NOW. The signal is confirmed clean at the HAT JST J9 pads (LA capture), but is not seen at the CrowPanel S3 IO19/IO20 pins. ESP-NOW is now the working data path, but the UART issue is unresolved.

### What Is Known

- HAT transmits clean UART frames every 5 s; Kingst LA1010 on J9 pin3/pin4 confirms the signal
- CrowPanel `PROBE [ms]` command (now removed from firmware) showed IO19 consistently HIGH with ≤3 transitions over 7 seconds — signal not arriving
- Physical wire swap (TX↔RX) at J8: no change
- Firmware TX/RX pin swap: no change
- Function-select switch positions tried: no change
- Continuity at J9 JST connector end: verified OK previously
- **Not yet done:** ohm test from JST UART1-OUT pads directly to S3 IO19/IO20 board pads (rules out PCB trace break)

### Recommended Next Steps

1. **Ohm test**: with board unpowered, measure continuity from CrowPanel JST UART1-OUT connector pads to the S3 IO19 and IO20 castellated/pad footprint. If open → PCB trace break or wrong connector pinout assumed.
2. **Try alternate S3 pins**: re-route `Serial1` to free GPIOs (e.g., IO4/IO5 or IO35/IO36) and bring those to test points. If signal appears → confirms IO19/IO20 are muxed/blocked on this board revision.
3. **Check V1.1 IO19/IO20 schematic**: Elecrow V1.1 wiki notes IO19/IO20 are shared with I2S MIC chip. Request/find V1.1 schematic to confirm mux topology and what controls it.
